# GaussDB 运维系统 — 常用命令速查

在项目根目录 `/Users/xieyuan/Code/GaussDBOM`（或你本机克隆路径）下执行；已配置虚拟环境时先 `source .venv/bin/activate`（路径按你实际为准）。

---

## 1. 环境与依赖

```bash
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

---

## 2. 数据库与检查

```bash
python manage.py migrate
python manage.py showmigrations
python manage.py check
```

若使用**生产配置**做部署前检查（需先设置环境变量，见 `.env.example`）：

```bash
export DJANGO_ENV=production
export DJANGO_SECRET_KEY='你的随机密钥'
export DJANGO_ALLOWED_HOSTS='example.com'
export DJANGO_BEHIND_HTTPS_PROXY=1
export DJANGO_CSRF_TRUSTED_ORIGINS='https://example.com'
python manage.py check --deploy
```

---

## 3. 启动开发服务

```bash
python manage.py runserver 0.0.0.0:8000
```

访问：`http://127.0.0.1:8000/`（未登录会跳转登录页）。

## 5. 管理员与用户（交互 / 脚本）

交互创建超级用户：

```bash
python manage.py createsuperuser
```

用 Shell 一次性设置密码（示例）：

```bash
python manage.py shell -c "
from apps.system.models import SysUser
u = SysUser.objects.get(username='admin')
u.set_password('YourPassword')
u.save()
"
```

## 6. 生产构建与运行（简要）

```bash
export DJANGO_ENV=production
# 按 .env.example 设置 SECRET_KEY、ALLOWED_HOSTS、数据库等

python manage.py migrate --noinput
python manage.py collectstatic --noinput
gunicorn gaussdbom.wsgi:application -c deploy/gunicorn.conf.py
```

健康检查（无需登录）：

- `GET /healthz/` — 存活  
- `GET /readyz/` — 数据库就绪

## 7. 环境变量说明

模板见项目根目录 **`.env.example`**；生产化说明见 **`design/生产化收口清单.md`**。

常用变量概要：

| 变量 | 含义 |
|------|------|
| `DJANGO_ENV` | `development`（默认）或 `production` |
| `DJANGO_SECRET_KEY` | 密钥；生产必填 |
| `DJANGO_ALLOWED_HOSTS` | 逗号分隔主机名 |
| `DJANGO_BEHIND_HTTPS_PROXY` | `1` 时在反向代理后启用 HTTPS 相关安全项 |
| `DJANGO_CSRF_TRUSTED_ORIGINS` | 可信前端源，如 `https://your.domain.com` |
| `DJANGO_DB_ENGINE` | `sqlite3` 或 `postgresql` |
| `DJANGO_LOG_LEVEL` | 如 `INFO`、`DEBUG` |
