针对HCS提单->问题审核阶段的进一步解释：
现有实现逻辑大致如下：

1 -> 2

|____ BU人员提单 && 组别:公有云 这一部分不涉及管控

|      |____ 周末/节假日/夜班 走排班表 不包含BU

|      |____ 工作日：局点大循环

|____ 临时提单：

|      |____ 周末/节假日/夜班 走排班表 管控从管控值班表拉人，非管控从内核值班表拉人

|      |____ 工作日：问题类型：管控/非管控

|      |     |____ 管控问题：管控值班表，如果没找到，从大循环里面找，（再找不到，设置成提单人）

|      |     |____ 非管控问题 从局点找合适处理人

|      |     |____ 局点找不到，就分配到大循环里面
​
其中，运维人员可以进行自提单，自提单直接到运维人员分析阶段，不涉及工单流转

这部分流程较为复杂，白班/晚班/节假日的流程有所不同，不同提单人提单后分单逻辑也略有差异：

晚班/节假日：走值班表，分为内核值班表和管控值班表，大体逻辑是类似的，内核问题在内核值班表内轮询分配，管控问题在管控值班表轮询分配。但是TAC提单后续不太能区分内核/管控，如果内核接到管控问题需要流转给管控，内核继续接单
白班：走大循环，分为内核大循环、管控大循环、公有云大循环，也是轮询分配

根据提单的角色，也有差异：
BU人员：仅处理公有云问题，提单后仅由维护团队的维护人员分析处理，BU人员不处理
HCS局点人员：处理HCS的内核问题和管控问题，管控问题维护人员不处理，转交给BU人员处理

针对问题审核的内部流转的解释
问题流转的几种方式：

专项走单，选择提交其他运维人员分析时，可以选择对应专项，从而根据专项表轮询人员进行处理，对应专项无人，则不可走单
其他，选择其他时，可以指定人员进行走单
管控问题，直接流转给管控人员
这一部分要处理的是，转单后，转单人需要额外接1个单子，被转单人需要更新接单时间（针对管控、专项）或者跳过下一次接单（指定人员走单）

问题字段配置
目前有7个问题流转阶段，每个问题阶段，都有不同的要填写的内容，并且，部分内容需要在多个阶段联动呈现。

支持多种字段类型（文本、日期、富文本、下拉选择、doer标注数据等）
下拉选择，需要支持配置可选值
部分字段的是否显示、是否必填，依赖于其他字段，如咨询问题有些字段不必填
