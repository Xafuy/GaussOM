# GaussDB 运维系统需求文档（当前版）

## 1. 用户与权限

### 1.1 用户与身份

- 支持普通用户与管理员（后台管理入口）
- 身份维度：`管控 / BU / 运维 / 开发 / 访客`
- 预留 W3 接入

### 1.2 角色与权限

- 支持角色与阶段处理权限可配置（`StagePermissionRule`）
- 工单前台操作以“阶段处理权限 + 当前处理人”联合约束：
  - 仅当前处理人可推进 / 转派 / 编辑字段 / 回退
  - 目标处理人必须具备目标阶段处理权限
- 看板入口按登录可见（不依赖角色权限中的 Permission 模块）

## 2. 值班与排班

### 2.1 轮值表（工作日白天）

- 支持多张轮值表
- 展示成员状态：在线/忙碌/离线
- 支持顺序与负载信息，用于分单

### 2.2 值班表（晚间/节假日）

- 支持多张值班表
- 以月历方式展示与维护每日值班人员
- 支持同日多名值班人员

### 2.3 请假

- 请假申请/审批/抄送
- 审批通过后分单自动跳过请假人员
- 请假结束后自动恢复

### 2.4 身份路由分单

- 分单时支持按提单人身份路由到不同轮值表/值班表
- 路由规则可配置（身份 + 时间窗口 + 优先级）

## 3. 运维单流程（核心）

八阶段：

1. HCS提单  
2. 问题审核  
3. 运维人员分析  
4. 开发人员分析  
5. 开发人员闭环  
6. 运维人员闭环  
7. 问题审核关闭  
8. 问题单关闭

状态流转（与设计文档一致）：

- HCS提单 ->（选择审核人）问题审核
- HCS提单 ->（撤回）撤销运维单
- 问题审核 ->（打回）HCS提单
- 问题审核 ->（值班系统分单）运维人员分析
- 运维人员分析 ->（同阶段分配）运维人员分析
- 运维人员分析 ->（指定用户）开发人员分析 / 开发人员闭环 / 运维人员闭环
- 开发人员分析 ->（指定用户）开发人员分析 / 开发人员闭环
- 开发人员闭环 ->（指定用户）运维人员闭环
- 运维人员闭环 ->（指定用户）问题审核关闭
- 问题审核关闭 -> 问题单关闭
- 问题审核关闭 ->（指定用户）运维人员闭环

补充规则：

- 运维人员支持自提单（直接进入运维人员分析）
- 除走向问题单关闭与撤销运维单外，其他流转必须有处理人
- 工单每个状态仅一个当前处理人，只有当前处理人可操作
- 回退时默认恢复上一阶段处理人（若可用）
- 工单详情顶部显示完整阶段进度（已完成/当前/未完成）

## 4. 分单设计（与值班系统）

### 4.1 触发原则

- 仅在工单进入“运维人员分析”阶段时触发值班分单：
  - 问题审核流转进入运维分析
  - 运维自提单直入运维分析
- 其他阶段处理人均由人工在流转时指定，不调用值班分单引擎

### 4.2 输入与规则

- 身份集合：默认取提单人身份（命中 `IdentityRouteRule`）
- 时间窗口：工作日白天优先轮值；夜间/节假日走值班
- 请假：自动跳过审批通过请假用户

### 4.3 失败策略

- 若值班系统无可分配人员，则本次流转失败并提示
- 工单保持原阶段，等待排班调整或人工介入

## 5. 阶段字段与联动

- 各阶段按字段模板渲染（文本/富文本/下拉）
- 支持字段联动显示、联动必填
- 详细字段来源：`design/问题单流程字段以及选项.txt`

## 6. 运维单列表默认展示字段

从左到右默认显示：

- 问题单号
- 当前阶段
- 起始日期
- 问题严重性
- 局点
- 实例简称
- 业务环境
- 当前处理人
- 问题描述
- 滞留时间
- SLA（按严重性计算）

列表视图口径：

- 我的待处理 / 我创建的 / 我协助的 / 全部

## 7. 统计分析（Dashboard）

- 提供多视图统计：人力投入、协助工单、问题归属、透传分析、SLA趋势、排班联动
- 支持时间窗口筛选（7/30/90天与自定义）
- 支持登录态控制（已登录可访问看板）

## 8. 当前实现边界说明

已实现：流程、分单、排班、月历、阶段权限、字段联动、统计看板、演示数据。  
未完全实现：配置中心全量 UI、复杂报表钻取、Doer 深度集成等。
