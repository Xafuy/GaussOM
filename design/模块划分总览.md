# GaussDB运维系统 模块划分总览

## 1. 目标

基于 `需求文档.md`，将系统拆分为可独立设计、开发、测试、迭代的模块，形成端到端闭环能力：
- 运维单全生命周期管理
- 值班与自动分单
- 人员组织与权限治理
- 配置化字段与流程
- 数据分析与可视化

## 3. 端到端主流程（高层）

1. 提单人发起工单（HCS/BU/临时/自提单）  
2. 系统依据班次、角色、问题类型执行自动分单  
3. 工单进入多阶段流转（审核、运维分析、开发分析、闭环、终审关闭）  
4. 阶段表单按配置渲染并校验（显示/必填/联动）  
5. 过程数据进入统计模型，支撑看板与复盘  
6. 管理员通过配置中心持续调整策略（流程、字段、轮询、模板、版本等）

## 4. 关键设计原则

- 配置优先：避免硬编码流程、字段、轮询规则
- 可追溯：所有分单与流转操作可审计
- 可回放：可复盘某工单在某时刻为何分配给某人
- 低耦合：流程引擎、分单引擎、字段引擎解耦
- 渐进演进：先支持核心流程，再逐步扩展复杂策略

## 5. 里程碑建议

- M1：组织权限 + 提单到问题审核 + 基础分单
- M2：完整7阶段流转 + 字段配置化 + 值班请假联动
- M3：专项/管控/指定转单机制 + Dashboard一期
- M4：策略优化、性能治理、审计报表与运营能力
