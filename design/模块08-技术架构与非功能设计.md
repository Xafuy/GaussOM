# 模块08：技术架构与非功能设计

## 1. 技术栈边界

- 后端框架：Django
- 数据库：SQLite（当前阶段）
- 前端：可先采用 Django Template，后续可演进前后端分离

## 2. 分层架构建议

- 接口层：REST API / 页面视图
- 应用层：工单、分单、排班、字段、统计服务
- 领域层：状态机、规则引擎、权限模型
- 基础层：ORM、缓存、任务调度、审计日志

## 3. 关键横切能力

- 认证鉴权：登录态、角色授权、接口鉴权
- 审计追踪：关键操作全链路日志
- 消息通知：待办提醒、超时提醒、审批提醒
- 异常处理：统一错误码与可读错误提示

## 4. 非功能要求

- 可用性：核心提单和分单流程高可用
- 性能：常用查询秒级，分单决策毫秒级
- 一致性：流程状态与分单记录强一致
- 可维护性：规则配置化、模块解耦、测试可覆盖
- 安全性：敏感字段脱敏、最小权限原则

## 5. 测试策略

- 单元测试：状态机、分单规则、字段联动规则
- 集成测试：提单到关闭全流程
- 回归测试：不同班次与角色的分单路径
- 数据校验：Dashboard指标与源数据对账

## 6. 演进建议

- 阶段1：SQLite + 单体服务快速落地核心流程
- 阶段2：引入缓存和异步任务，提升统计与通知能力
- 阶段3：按模块拆分服务边界，支持更大规模并发
