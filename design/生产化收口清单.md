# GaussDB 运维系统 — 生产化收口清单

本文对应代码已落地项（含文件路径），便于上线评审与巡检。

## 1. 配置与环境隔离

| 项 | 状态 | 说明 |
|----|------|------|
| 多环境 settings | 已落地 | `gaussdbom/settings/__init__.py` 按 `DJANGO_ENV` 加载 `development` / `production` |
| 敏感配置外置 | 已落地 | `DJANGO_SECRET_KEY`、主机、数据库、HTTPS 相关均由环境变量注入（见 `.env.example`） |
| 开发密钥默认值 | 已落地 | 仅 `development` 在缺少密钥时使用不安全占位（勿用于生产） |
| 生产强制校验 | 已落地 | `production` 下要求 `DJANGO_ALLOWED_HOSTS`；默认校验 `SECRET_KEY` 强度（可用 `DJANGO_SECRET_KEY_STRENGTH_CHECK=0` 临时关闭内网场景） |

## 2. 安全

| 项 | 状态 | 说明 |
|----|------|------|
| DEBUG 默认关闭（生产） | 已落地 | `production.DEBUG` 仅当 `DJANGO_DEBUG=1` 时开启 |
| 安全响应头 / XSS / MIME | 已落地 | `SECURE_BROWSER_XSS_FILTER`、`SECURE_CONTENT_TYPE_NOSNIFF`、`X_FRAME_OPTIONS` |
| HTTPS 与 HSTS | 已落地 | `DJANGO_BEHIND_HTTPS_PROXY=1` 时启用 `SECURE_PROXY_SSL_HEADER`、安全 Cookie、`SECURE_SSL_REDIRECT`、`HSTS`（可调） |
| CSRF 可信源 | 已落地 | `DJANGO_CSRF_TRUSTED_ORIGINS`（逗号分隔，须含协议，如 `https://a.com`） |

## 3. 静态资源与 WSGI

| 项 | 状态 | 说明 |
|----|------|------|
| WhiteNoise | 已落地 | 生产 `MIDDLEWARE` 插入 WhiteNoise；`STORAGES["staticfiles"]` 使用压缩静态 |
| collectstatic | 需部署执行 | 生产构建：`python manage.py collectstatic --noinput`，产物目录 `staticfiles/`（已加入 `.gitignore`） |
| Gunicorn | 已落地 | `requirements.txt` 含 `gunicorn`；示例配置 `deploy/gunicorn.conf.py` |

## 4. 数据库

| 项 | 状态 | 说明 |
|----|------|------|
| SQLite 默认 | 已落地 | 开发可用；路径 `DJANGO_SQLITE_PATH` 可指定 |
| PostgreSQL | 已落地 | `DJANGO_DB_ENGINE=postgresql` + `DJANGO_DB_NAME` 等；依赖 `psycopg2-binary` |

## 5. 可观测性与运维

| 项 | 状态 | 说明 |
|----|------|------|
| 结构化日志 | 已落地 | `LOGGING` 输出至 stdout（容器/K8s 友好），级别 `DJANGO_LOG_LEVEL` |
| 存活探针 | 已落地 | `GET /healthz/` 不连库 |
| 就绪探针 | 已落地 | `GET /readyz/` 校验数据库连接 |

## 6. 部署命令备忘

```bash
export DJANGO_ENV=production
export DJANGO_SECRET_KEY='请使用高强度随机串'
export DJANGO_ALLOWED_HOSTS='your.domain.com'
export DJANGO_BEHIND_HTTPS_PROXY=1
export DJANGO_CSRF_TRUSTED_ORIGINS='https://your.domain.com'

python manage.py migrate --noinput
python manage.py collectstatic --noinput
gunicorn gaussdbom.wsgi:application -c deploy/gunicorn.conf.py
```

上线前建议：

```bash
DJANGO_ENV=production ... python manage.py check --deploy
```
