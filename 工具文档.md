# GaussDBOM 工具文档

## 1. 本地运行

```bash
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

访问地址：`http://127.0.0.1:8000/`

**Windows：** 若曾遇到 `SQLite 不支持 JSONField` 的迁移错误，请拉取最新代码：阶段字段已使用 `JSONTextField`，不依赖 SQLite JSON1；或使用 Python 3.10+。

---

## 2. 创建超级用户

用于首次登录 Django 管理后台（`/admin/`）或需要独立管理员账号时，在项目根目录执行：

```bash
python manage.py createsuperuser
```

按提示输入用户名、邮箱（可跳过）、密码（输入两次）。需先完成 `python manage.py migrate`。

---

## 3. 演示数据生成

项目内置了演示数据种子命令：

```bash
python manage.py seed_demo_data
```

命令特点：

- 幂等，可重复执行；
- 自动生成演示用户、角色绑定、轮值/值班/请假、演示工单；
- 便于测试“自动分单、请假跳过、工单流转、RBAC权限控制”。

执行完成后会输出：

- `bootstrap_gaussdbom 完成。`
- 演示账号与工单范围提示。

## 5. 演示数据内容

`seed_demo_data` 默认生成以下数据：

- 角色绑定：`pl`、`ops`、`dev`、`control`、`bu`、`guest` 体系下的示例用户关系；
- 排班数据：
  - 多张白班轮值表（默认、BU、管控、公有云）；
  - 多张晚间/节假日值班表（默认、BU、管控、内核，含当天与次日安排）；
  - 1 条已审批请假（用于验证分单跳过请假人员）；
- 路由数据：
  - 已生成 BU / 管控在白天与晚间的身份分单路由规则（可在后台继续改）；
- 工单数据：`OM-DEMO-0001` 到 `OM-DEMO-0006`，覆盖多个阶段。
- 看板数据：工单时间分布与字段归属已拉开，可直接在 Dashboard 看到趋势与分类图。

值班管理页面补充：

- 值班表已按**月历**展示；
- 可切换不同值班表；
- 可点击某一天查看并编辑当日排班（仅管理员可编辑，非管理员只读）。

## 6. RBAC 关键点

系统默认包含以下权限码：

- 阶段处理权限：按 `StagePermissionRule` 配置用户可处理的阶段
- 前台操作约束：仅“当前处理人”可推进 / 转派 / 编辑字段 / 回退
- 提单权限：需具备 `HCS提单` 阶段处理能力
- 统计看板：登录用户可查看

默认策略：

- `staff/superuser` 默认具备阶段处理能力；
- 非 `staff/superuser` 需在后台绑定角色，前台才显示对应阶段按钮；
- 角色是否可操作某阶段，以阶段处理权限规则为准（不再依赖 `ticket:*` 权限点）。
- 运维单处理默认仅“当前处理人”可操作（前台无管理员例外）。
- 问题审核阶段仅允许两种流转：`打回HCS提单` / `进入运维人员分析（值班系统分单）`。
